# 05 — AI Stratejisi & Mimarisi

**Status:** Active | **Last Updated:** 2026-05-18 | **Owner:** Erdi

> İnşaat Borsam'ın **kalbi AI**. Marka pozisyonu "yapay zeka destekli inşaat tedarik borsası." Bu dosya her AI çağrısının nerede, nasıl, neden olduğunu tanımlar.

---

## Felsefe

### 1. AI Yardımcıdır, Patron Değildir
- AI satıcı seçmez, AI fiyat empoze etmez
- AI sadece **öneri** sunar, son karar her zaman kullanıcıdadır
- Şeffaflık: kullanıcı "bu öneriyi neden aldım" sorgusuna cevap görebilmeli

### 2. Doğru Model, Doğru İş
- Basit görev → ucuz model (Haiku 4.5, Gemini Flash)
- Karmaşık görev → güçlü model (Sonnet 4.7)
- Maliyet kontrolü asla göz ardı edilmez

### 3. Streaming UX
- Kullanıcı beklediğinde, AI **canlı yazmalı**
- "Düşünüyor..." spinner yerine streaming response
- Anthropic SDK streaming kullan

### 4. Failure Gracefully
- AI cevap veremezse, manuel akış devreye girer
- Hiçbir kritik akış sadece AI'a bağımlı değildir
- Timeout: 10 saniye, sonra fallback

### 5. Veriden Öğren, Veriyi Sakla
- Her AI çağrısı log'lanır (anonimleştirilmiş)
- Sonuç kullanıcı tarafından kabul/ret edildi mi?
- Bu veriler ileride fine-tuning için altın değerinde

## MVP'deki AI Özellikleri (Faz 1)

İlk lansman'da **sadece 4 AI özelliği** aktif:

### 1. RFQ Parser (Teklif Talebi Çözümleyici)

**Ne:** Alıcı serbest metinle RFQ yazınca, AI yapılandırılmış veriye dönüştürür.

**Örnek:**
> Input: "120 metrekare 60x60 granit seramik lazım, açık gri ton, İstanbul Anadolu yakası, 2 hafta içinde teslim"
> 
> Output:
> ```json
> {
>   "category": "ceramic_floor",
>   "quantity": 120,
>   "unit": "m²",
>   "specifications": {
>     "size": "60x60",
>     "type": "granit",
>     "color": "açık gri"
>   },
>   "delivery_city": "İstanbul",
>   "delivery_region": "Anadolu yakası",
>   "delivery_deadline_days": 14,
>   "extracted_brand": null,
>   "confidence": 0.92
> }
> ```

**Model:** Claude **Haiku 4.5** (ucuz, hızlı, JSON için yeterli)

**Yer:** Supabase Edge Function `rfq-parser/`

**Maliyet:** ~$0.002 per call (ortalama 300 input tokens, 200 output)

**Akış:**
1. Alıcı RFQ formunu doldurur
2. Edge Function çağrılır
3. Haiku ile parse edilir
4. `rfqs.parsed_data` kolonuna kaydedilir
5. Seller-match için kullanılır

**Prompt:** `packages/ai/prompts/rfq_parser.md` dosyasında. Kısaca:
```
Sen Türk inşaat sektörü için uzmansın. Verilen RFQ metnini yapılandırılmış JSON'a çevir.

Kurallar:
- Sadece açıkça belirtilen bilgileri çıkar, tahmin yapma
- Belirsizse `null` döndür
- Confidence skoru ekle (0.0-1.0)
- Türkçe ölçü birimleri kullan: m², ton, adet, kg, paket

Çıktı formatı: [şema]
```

### 2. Ürün Kategorize (Auto-Categorization)

**Ne:** Satıcı yeni ürün ekleyince, AI doğru kategoriyi önerir.

**Model:** Claude **Haiku 4.5** + **Vision** (Gemini Flash da olur, daha ucuz)

**Yer:** Edge Function `product-categorize/`

**Akış:**
1. Satıcı ürün fotoğrafı yükler + isim girer
2. Edge Function çağrılır
3. AI: "Bu ürün hangi kategoride? Hangi alt kategoride?"
4. Confidence > 0.85 ise otomatik atar
5. Düşükse satıcıya seçenek sunar

**Maliyet:** ~$0.005 per call (vision pahalı, ama nadir)

**Prompt:** Kategori listesini bağlam olarak ver:
```
Bu ürün fotoğrafı ve "{product_name}" ismi için aşağıdaki kategori 
listesinden en uygun 3 seçeneği bul.

Mevcut kategoriler: [JSON tree]

Her seçenek için confidence skor ver.

Çıktı: [seç şema]
```

---

### 3. Semantic Search (Anlamsal Arama)

**Ne:** "Beyaz banyo seramiği yumuşak doku" gibi doğal dil aramaları **gerçek sonuç** versin.

**Teknoloji:** 
- **OpenAI text-embedding-3-small** (1536 dim, ucuz: $0.02 per 1M tokens)
- **pgvector HNSW index** Supabase'de

**Yer:** 
- Embedding üretimi: Edge Function `generate-embedding/`
- Arama: Supabase RPC `search_products_semantic`

**Akış:**
1. Satıcı ürün ekler → Edge Function ürünün embedding'ini üretir → `products.embedding` kolonuna kaydeder
2. Alıcı arama yapar → query embedding üretilir → pgvector ile en yakın ürünler bulunur
3. Top 20 sonuç → ek filtreler (lokasyon, fiyat, satıcı) → kullanıcıya gösterilir

**Maliyet:** Embedding üretimi neredeyse bedava. Query başına ~$0.0001.

**Pratik notlar:**
- Embedding üretimi async — yeni ürün eklendikten 5-10 saniye sonra aranabilir
- Hibrit search: vektör + keyword (PostgreSQL FTS) → daha iyi sonuç
- Periodik re-embedding: model güncellendiğinde tüm embeddings yenilenir

### 4. Satıcı Eşleştirme (RFQ → Sellers)

**Ne:** Alıcı RFQ açtığında, AI en uygun 10 satıcıyı seçip onlara yollar.

**Model:** Custom skorlama (AI değil aslında, ama "akıllı"). Faz 2'de Sonnet 4.7 ile gelişmiş eşleştirme.

**Algoritma (Faz 1):**
```
score = 
  (cosine_similarity(rfq.embedding, seller.products_avg_embedding) * 0.50) +
  (category_match ? 0.30 : 0) +
  (seller.rating_avg / 5.0 * 0.10) +
  (seller.is_verified ? 0.05 : 0) +
  (distance_factor * 0.05)
```

**Yer:** Supabase RPC `match_sellers_for_rfq`

**Faz 2 yükseltme:** Sonnet 4.7 ile **açıklayıcı eşleştirme** — "Bu satıcıyı seçtim çünkü 3 ay önce benzer projede başarılı oldu, stok mevcut, lokasyon uygun."

## Faz 2+ AI Özellikleri (Sıraya Göre)

### 5. AI Malzeme Listesi (BOM Generator)

**Ne:** Müteahhit proje planını yükler veya yazar, AI tam malzeme listesi çıkarır.

**Örnek:**
> Input: "100 daireli bir site, 4 katlı, 2+1 ortalama, kaba inşaat aşamasında"
> 
> Output: 
> - C30 hazır beton: 5,400 m³
> - S420 nervürlü demir: 540 ton
> - Tuğla (13x19x19): 850,000 adet
> - ... (50+ kalem)

**Model:** **Claude Sonnet 4.7** (karmaşık çıkarım gerektirir)

**Maliyet:** ~$0.10 per generation (3K input, 5K output)

**Yer:** Edge Function `bom-generator/` + Sonnet 4.7

**Eğitim:** Türk inşaat normlarına göre prompt zenginleştirilmiş — TS 500, TS EN normları referansla.

### 6. WhatsApp AI Asistan (Satıcı için)

**Ne:** Satıcının WhatsApp'ına gelen mesajları AI cevaplar (önce satıcının onayıyla).

**Akış:**
1. Müşteri WhatsApp'ta sorar: "Weber 25kg yapıştırıcı kaç para?"
2. n8n WhatsApp webhook → Edge Function
3. AI ürünleri/fiyatları bilir (Supabase'den çeker)
4. Cevap üretir: "Merhaba, Weber Kerafix 25kg'ın fiyatı ₺245. Kaç torba istediğinizi söylerseniz toplam fiyatı çıkarayım."
5. Satıcı onayı (Faz 2.1) veya otomatik gönder (Faz 2.2)

**Model:** **Claude Sonnet 4.7**

**Maliyet:** ~$0.03 per conversation turn

**Yer:** n8n workflow + Anthropic API çağrısı

### 7. Fiyat Anomali Tespiti (Admin için)

**Ne:** Sistem fiyat dalgalanmalarını izler, anomali bulunca admin'i uyarır.

**Örnek:**
> "İstanbul'da Weber yapıştırıcı 25kg ortalaması ₺245. Bayi X bunu ₺189'a satıyor. 3 ihtimal:
> 1) Stok eritiyor (kampanya?)
> 2) Kalite şüpheli
> 3) Fırsat fiyatı"

**Model:** İstatistik + Haiku 4.5 yorumlaması

**Yer:** n8n scheduled (saatlik) → Haiku ile insight üret → Admin Console'da bildirim

**Maliyet:** ~$0.01 per detected anomaly

---

### 8. Akıllı Müşteri Destek Chatbot

**Ne:** Buyer/Seller App içinde "Yardım" tuşuna basan kullanıcıyla AI sohbet eder. Tier 1 destek.

**Model:** **Claude Sonnet 4.6** (cost-effective, yeterli)

**Yer:** Edge Function `support-chat/` + Anthropic Streaming

**Çözebileceği:**
- "Siparişim nerede?" → Supabase query + cevap
- "Şifremi nasıl değiştiririm?" → adımlar
- "Komisyon nasıl çalışıyor?" → KB'den cevap
- "Refund nasıl alırım?" → policy + form yönlendirme

**Çözemediği:** Human agent'a escalate (conversation context'iyle birlikte).

## Prompt Mühendisliği Prensipleri

### 1. Sistem Promptu: Sektörel Uzmanlık
Tüm prompt'lar şu sistem mesajıyla başlar:

```
Sen Türkiye inşaat sektöründe uzman bir AI asistansın. 
İnşaat malzemesi tedariği, fiyatlandırma, lojistik ve sektör 
terminolojisi konusunda derin bilgiye sahipsin.

Yanıtlarında:
- Türkçe konuş (sektör jargonunu koru: "metraj", "hak ediş", "sevkiyat")
- Net ve kısa ol — müteahhitler süslü dil sevmez
- Belirsizse "bilmiyorum" de, tahmin yapma
- Para birimi varsa ₺ kullan
```

### 2. Few-Shot Examples
Her prompt 2-3 örnek içerir. Türkçe veriyle.

### 3. Structured Output
Mümkün olduğunda JSON çıktı iste. `response_format: json_object` veya Pydantic-style schema.

### 4. Chain of Thought (Düşünme Zinciri)
Karmaşık görevlerde Claude'a "önce düşün, sonra cevap ver" de:
```
Önce sorunu analiz et. Hangi bilgilere ihtiyacın var?
Sonra çözümü adım adım inşa et.
En son nihai JSON çıktısını ver.
```

### 5. Versiyonlama
Her prompt versiyonlanır: `packages/ai/prompts/v1/rfq_parser.md`. Eski versiyonlar saklanır (A/B test, rollback için).

## Edge Function Şablonu

Her AI Edge Function şu yapıdadır:

```typescript
// supabase/functions/rfq-parser/index.ts
import { serve } from "https://deno.land/std/http/server.ts";
import Anthropic from "npm:@anthropic-ai/sdk";

const anthropic = new Anthropic({
  apiKey: Deno.env.get("ANTHROPIC_API_KEY")!,
});

serve(async (req) => {
  // 1. Auth check
  const authHeader = req.headers.get("Authorization");
  if (!authHeader) return new Response("Unauthorized", { status: 401 });
  
  // 2. Rate limit check (Supabase RPC)
  const userId = await getUserIdFromAuth(authHeader);
  const allowed = await checkRateLimit(userId, "rfq-parser", 60); // 60/dk
  if (!allowed) return new Response("Too many requests", { status: 429 });
  
  // 3. Input validation (Zod)
  const body = await req.json();
  const validated = rfqParserSchema.parse(body);
  
  // 4. Claude çağrısı
  const response = await anthropic.messages.create({
    model: "claude-haiku-4-5-20251001",
    max_tokens: 500,
    system: SYSTEM_PROMPT,
    messages: [{ role: "user", content: validated.text }],
  });
  
  // 5. Parse + validate output
  const parsed = JSON.parse(response.content[0].text);
  const validatedOutput = rfqOutputSchema.parse(parsed);
  
  // 6. Log
  await logAICall({
    user_id: userId,
    function: "rfq-parser",
    model: "haiku-4.5",
    input_tokens: response.usage.input_tokens,
    output_tokens: response.usage.output_tokens,
    cost_usd: calculateCost(response.usage),
  });
  
  // 7. Return
  return new Response(JSON.stringify(validatedOutput), {
    headers: { "Content-Type": "application/json" },
  });
});
```

## Veri Privacy & AI

### Hangi Veriler AI'a Gönderilebilir?
- ✅ RFQ metni (alıcı yazdı, public görünür)
- ✅ Ürün ismi, açıklaması (satıcı yazdı, public)
- ✅ Anonim talep sıklığı, fiyat ortalamaları

### Hangi Veriler AI'a Gönderilmez?
- ❌ Telefon, email (PII)
- ❌ Vergi numarası, IBAN
- ❌ Kart bilgisi (zaten Iyzico/Stripe'da)
- ❌ Özel mesajlaşma içeriği (kullanıcı opt-in vermeden)
- ❌ Vergi levhası, kimlik fotoğrafı

## AI Çağrı Logging Tablosu

```sql
CREATE TABLE ai_call_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  user_id UUID REFERENCES profiles(id),
  function_name TEXT NOT NULL, -- 'rfq-parser', 'product-categorize'
  model TEXT NOT NULL,
  
  input_tokens INT,
  output_tokens INT,
  cost_usd NUMERIC(10,6),
  
  duration_ms INT,
  status TEXT CHECK (status IN ('success', 'error', 'timeout')),
  error_message TEXT,
  
  -- Hangi kullanıcı eylemi sonucu çağrıldı?
  triggered_by_resource_type TEXT, -- 'rfq', 'product'
  triggered_by_resource_id UUID,
  
  -- Sonuç kullanıcı tarafından kabul edildi mi?
  user_accepted_result BOOLEAN,
  
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_ai_logs_function ON ai_call_logs(function_name, created_at DESC);
CREATE INDEX idx_ai_logs_user ON ai_call_logs(user_id);
```

Bu tablo:
- Maliyet takibi için
- A/B test için (model değişikliğinin etkisi)
- Fine-tuning veri seti için (gelecek)

## AI Quality Assurance

### Pre-launch Test Suite
Her AI özelliği için **test seti** oluştur:
- 50+ Türkçe örnek input
- Beklenen output template
- Edge case'ler (boş input, çok uzun, Türkçe karakter, çoklu ürün)

### A/B Testing (Faz 2)
- Aynı prompt'un 2 versiyonu canlıda %50-%50
- 1000 çağrı sonra sonuç karşılaştır
- Daha iyisi kalır

### User Feedback
Her AI çıktısının yanında **👍 / 👎** butonu:
- 👍: prompt iyi çalışıyor
- 👎: fine-tuning datası

### Hallucination Önleme
- Claude'a "kesin bilmediğin şeyi söyleme" diye explicit talimat
- JSON output validation: çıktı şemada değilse retry
- Confidence skor: 0.8+ ise göster, altındaysa kullanıcıya "AI emin değil" uyarısı
